~~~
npm start
개발 모드로 앱을 실행합니다.
브라우저에서 http://localhost:3000 을 열어 확인할 수 있습니다.
코드를 수정하면 페이지가 자동으로 새로고침됩니다.
콘솔에서 린트 오류도 확인할 수 있습니다.


npm test
대화형 감시 모드로 테스트 러너를 실행합니다.
테스트 실행에 대한 자세한 정보는 관련 문서를 참고하세요.


npm run build
프로덕션용 앱을 build 폴더에 빌드합니다.
프로덕션 모드에서 React를 올바르게 번들링하고 최적의 성능을 위해 빌드를 최적화합니다.
빌드된 파일은 최소화되며 파일 이름에 해시가 포함됩니다.
이제 앱을 배포할 준비가 되었습니다!


npm run eject
주의: 이 작업은 되돌릴 수 없습니다. eject를 실행하면 돌이킬 수 없습니다!
빌드 도구와 설정 선택에 만족하지 않는다면 언제든 eject를 실행할 수 있습니다. 이 명령은 프로젝트에서 단일 빌드 종속성을 제거합니다.
대신 모든 설정 파일과 종속성(webpack, Babel, ESLint 등)을 프로젝트로 직접 복사하여 완전히 제어할 수 있게 됩니다.
 eject를 제외한 모든 명령은 계속 작동하지만, 복사된 스크립트를 가리키게 되므로 이를 조정할 수 있습니다. 이 시점부터는 모든 것을 직접 설정해야 합니다.
eject를 반드시 사용할 필요는 없습니다. 이 기능은 중소규모 배포에 적합하며, 이 기능을 사용해야 한다는 부담을 가질 필요는 없습니다.
 하지만 필요할 때 커스터마이징할 준비가 되어 있다는 점에서 유용한 도구입니다.
